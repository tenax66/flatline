# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

FLATLINE (https://tanka.cc) is a Jekyll 4.4 site for an internet tanka (短歌) group. Members' tankas are published roughly monthly as posts. Content and UI text are in Japanese.

## Commands

```sh
bundle install                  # install gems
bundle exec jekyll serve        # local dev server at http://localhost:4000
bundle exec jekyll build        # build to _site
npx -y pagefind                 # build search index (run AFTER jekyll build; config in pagefind.yml)
```

There are no tests or linters. Deployment is automatic: pushing to `main` (or `preview/**`, which builds with `--future`) triggers `.github/workflows/publish.yml`, which builds Jekyll + Pagefind and deploys `_site` to Cloudflare Pages.

## Architecture

### Content model

- `_posts/` — one file per author per monthly issue (~550 files). Front matter: `title`, `author`, `category` (same as author), `emoji`, `image`, `date`. Posts default to `layout: post` via `_config.yml` defaults. Permalinks are `/:title/` (the filename slug).
- Post body convention: tanka wrapped in `<div class="tanka-area"><div class="tanka">…` (styled display), followed by `---` and a `<details>` block with a plain-text fallback of the same poems.
- `_pages/` — standalone pages (issue indexes like `flatline-N.md`, events, etc.), default `layout: page`.
- Author pages are generated by `jekyll-archives` at `/category/:name/`. Category slugs for Japanese author names come from the custom Liquid filter `slugify_author` in `_plugins/slugify_author.rb`; a `custom_author` front matter field can override the name used for the slug (see `_layouts/post.html`).

### Content generation scripts

`chore/` (excluded from the site) holds Python helpers fed by `input.txt`:
- `chore/generate_tanka_pages.py` — splits a batch submission text into per-author post files; contains the author→emoji map.
- `chore/generate_month_page.py` — generates the monthly summary post (`author: flatline`).

### Styling and frontend

- Bootstrap 5.3 from CDN + custom styles: `assets/css/styles.scss` imports `_sass/custom.scss` and `_sass/main.scss`.
- **`DESIGN.md` is the design guide — read it before any styling/UI work.** Key rules: paper-like aesthetic (no glow/blur/gradients/shadows), hairline rules, terracotta accent, light theme = drafting paper, dark theme = cyanotype; tanka body keeps pixel fonts (lores-12 / dotgothic16, loaded via Adobe Fonts).
- Light/dark theme via Bootstrap's `data-bs-theme`, toggled by `assets/js/color-modes.js`.
- `assets/js/post-swipe-nav.js` enables swipe navigation between posts (`#postSwipeRoot` in `_layouts/post.html`).
- Search is Pagefind (`force_language: ja`); the UI is loaded in `_layouts/default.html`.

### Notes

- `chore/`, `DESIGN.md`, `README.md` are excluded from the built site via `_config.yml`.
- Pagination: 40 posts per page at `/:num/`.
